## 推荐问题模板

### 概念理解类
```
- 用简单的语言解释 [概念]
- [概念 A] 和 [概念 B] 有什么区别？
- 能否用一个实际例子说明 [概念]？
- 这个概念在 TK 运营场景中如何应用？
```

### 代码实践类
```
- 如何用 LangChain 实现 [功能]？
- 这段代码的作用是什么？[粘贴代码]
- 帮我优化这段提示词：[粘贴提示词]
- 如何用 OpenClaw 调用 [模型]？
```

### 问题解决类
```
- 我遇到了 [错误信息]，可能是什么原因？
- [任务] 的最佳实践是什么？
- 如何评估 [模型/方法] 的效果？
- 对比 [方案 A] 和 [方案 B] 的优缺点
```

### 复习总结类
```
- 总结本周学习的核心概念
- 列出 [主题] 的关键知识点
- 生成一份 [主题] 的复习大纲
- 我学过的 [主题] 相关内容有哪些？
```

## 每周工作流

### 周一（新知识导入）
1. 上传本周要学习的课程资料
2. 创建新的学习主题
3. 生成学习计划问题列表

### 每日（学习记录）
1. 从 Notion 导入当日学习笔记
2. 向 NotebookLM 提问不理解的概念
3. 保存关键问答到笔记

### 周五（周复盘）
1. 提问："总结本周学习的所有内容"
2. 提问："列出我还不太理解的概念"
3. 生成周复习大纲
4. 导出复习材料到 Notion

### 月末（阶段总结）
1. 提问："生成阶段 1 的完整知识图谱"
2. 提问："列出所有实践项目和代码示例"
3. 导出阶段总结报告

## 知识库组织技巧

### 1. 使用标签
在上传资料时，在标题中添加标签：
- `[课程]` - 课程资料
- `[论文]` - 学术论文
- `[笔记]` - 个人笔记
- `[代码]` - 代码示例
- `[文档]` - 工具文档

### 2. 定期清理
- 删除过时的草稿笔记
- 合并重复的资料
- 归档已完成的阶段资料

### 3. 建立索引
创建一个"知识库索引"文档，记录：
- 所有上传资料的列表
- 重要问答的链接
- 常用问题模板

## 与 Notion 的集成

### 双向同步流程

```
Notion (任务管理)          NotebookLM (知识管理)
     │                           │
     │  1. 创建学习任务           │
     │────────────────────────>  │
     │                           │
     │  2. 上传资料到知识库        │
     │<────────────────────────  │
     │                           │
     │  3. 学习并记录笔记          │
     │────────────────────────>  │
     │                           │
     │  4. 提问获取解答           │
     │<────────────────────────  │
     │                           │
     │  5. 导出笔记回 Notion      │
     │────────────────────────>  │
     │                           │
     │  6. 更新任务状态           │
     │                           │
```

```markdown
## 🤖 核心问答

**A:** [答案摘要]

## 💡 关键收获
-

```
