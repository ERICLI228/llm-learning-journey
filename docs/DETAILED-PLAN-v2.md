## 🎓 NotebookLM 使用指南（详细）

### 1. 资料上传

#### 推荐上传的资料类型
| 资料类型 | 格式 | 说明 |
|----------|------|------|
| 课程视频字幕 | TXT/Markdown | 下载或复制字幕文本 |
| 论文 PDF | PDF | 核心论文全文 |
| 技术博客 | URL/复制文本 | 高质量技术文章 |
| 代码片段 | TXT/Markdown | 关键代码和注释 |
| 实验输出 | TXT/Markdown | 模型响应示例 |
| 个人笔记草稿 | TXT/Markdown | 初步理解记录 |

#### 上传时机
- **每天学习前**: 上传当日学习资料
- **实验完成后**: 上传实验输出和日志
- **每周复盘前**: 上传本周所有笔记

---

### 2. 高效提问模板

#### 概念理解类
```
请用简单的语言解释 [概念]
请用比喻解释 [概念]
[概念 A] 和 [概念 B] 有什么区别？
这个概念在 [实际场景] 中如何应用？
```

#### 代码理解类
```
这段代码的作用是什么？[粘贴代码]
有没有更优的实现方式？
这段代码的关键步骤是什么？
如何调试这段代码？
```

#### 对比分析类
```
对比 [方案 A] 和 [方案 B] 的优缺点
[模型 A] 和 [模型 B] 在 [任务] 上各有什么优劣？
基于我的实验数据，分析两者的差异
```

#### 错误排查类
```
我遇到了以下错误：[错误信息]
相关代码是：[代码]
可能的原因有哪些？
如何解决？
```

#### 学习材料生成类
```
根据我上传的所有笔记，生成一份知识点总结
生成一份 [主题] 的面试题库
生成一份 [主题] 的复习大纲
用思维导图的形式总结 [主题]
```

#### 项目辅助类
```
检查我的系统设计文档，指出遗漏的部分
我的 [模块] 在扩展性方面有哪些潜在问题？
如何优化 [指标]？
生成一份技术博客的亮点总结
```

---

### 3. NotebookLM + Notion 联动流程

```
1. 在 Notion 中创建学习任务
   ↓
2. 在 NotebookLM 中上传相关资料
   ↓
3. 学习并向 NotebookLM 提问
   ↓
4. 复制 NotebookLM 的优质回答到 Notion 笔记
   ↓
5. 在 Notion 中更新任务状态
   ↓
6. 周末让 NotebookLM 生成本周总结
   ↓
7. 将总结复制到 Notion 周复盘页面
```
