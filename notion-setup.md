# 📓 Notion 学习数据库设置指南

## 数据库结构

### 1. 主数据库：🚀 大模型学习计划

**创建位置**: Notion 工作区 → 新建数据库 → Table View

#### 字段配置

| 字段名 | 类型 | 说明 | 选项值 |
|--------|------|------|--------|
| 任务 ID | Title | 如 1.1, 2.3, 3.5 | - |
| 任务名称 | Text | 任务简短描述 | - |
| 所属阶段 | Select | 阶段 1/2/3 | 阶段 1, 阶段 2, 阶段 3 |
| 周次 | Select | W1-W12 | W1, W2, ..., W12 |
| 计划开始 | Date | 计划开始日期 | - |
| 计划结束 | Date | 计划结束日期 | - |
| 实际开始 | Date | 实际开始日期 | - |
| 实际结束 | Date | 实际结束日期 | - |
| 状态 | Status | 任务状态 | ⏳ 待开始，🔥 进行中，✅ 已完成，❌ 已跳过 |
| 优先级 | Select | 任务优先级 | 🔴 高，🟡 中，🟢 低 |
| 预计耗时 | Number | 预计小时数 | - |
| 实际耗时 | Number | 实际小时数 | - |
| 使用工具 | Multi-select | 涉及的工具 | OpenClaw, Ollama, Gemini, Qwen, Copilot, LangChain, NotebookLM |
| 笔记链接 | Relation | 关联到笔记数据库 | - |
| 代码链接 | URL | GitHub 仓库或代码片段 | - |
| 验收结果 | Text | 验收是否通过及说明 | - |
| 遇到的问题 | Text | 遇到的困难和解决方案 | - |

#### 视图配置

**1. 看板视图 (By Status)**
- 按状态分组
- 卡片显示：任务 ID, 任务名称，优先级，所属阶段

**2. 时间线视图 (Timeline)**
- 按计划开始/结束日期显示
- 用于查看整体进度

**3. 阶段视图 (By Phase)**
- 按所属阶段分组
- 用于阶段复盘

**4. 本周任务 (This Week)**
- 筛选：计划开始日期 = 本周
- 用于每日查看

### 2. 笔记数据库：📝 学习笔记

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 标题 | Title | 笔记标题 |
| 日期 | Date | 创建日期 |

| 关联任务 | Relation | 关联到主数据库 |
| 标签 | Multi-select | Transformer, Attention, RAG, LoRA, Agent, etc. |
| NotebookLM 链接 | URL | 对应的 NotebookLM 知识库链接 |

## 每日工作流

### 早上（计划）
1. 打开 Notion → 查看"本周任务"视图
2. 确认今日要完成的任务
3. 更新任务状态为"🔥 进行中"

### 晚上（记录）
1. 在笔记数据库创建今日学习记录
2. 更新完成任务的状态为"✅ 已完成"
3. 填写实际耗时和验收结果
4. 记录遇到的问题和解决方案

### 周末（复盘）
1. 切换到"看板视图"查看整体进度
2. 创建周复盘笔记
3. 调整下周计划
